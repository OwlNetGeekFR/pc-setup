# Politique de sécurité

## Versions prises en charge

Seule la dernière version stable publiée bénéficie des correctifs de sécurité.

## Signaler une vulnérabilité

N’ouvrez pas d’Issue publique contenant les détails d’une vulnérabilité.

Utilisez le bouton **Report a vulnerability** dans l’onglet **Security** du dépôt afin de transmettre un rapport privé. Indiquez si possible :

- la version concernée ;
- les étapes de reproduction ;
- l’impact estimé ;
- les journaux expurgés de toute donnée personnelle ;
- une proposition de correction, si vous en avez une.

Un accusé de réception sera fourni dès que possible. La vulnérabilité ne doit être rendue publique qu’après la disponibilité d’un correctif ou avec l’accord du mainteneur.

## Téléchargements officiels

Les exécutables officiels sont distribués uniquement dans les [Releases GitHub](https://github.com/OwlNetGeekFR/OwlSetup/releases). Chaque Release fournit un fichier `SHA256.txt` permettant de vérifier l’intégrité des téléchargements.
